# Database ownership

| 영역 | 주 담당 | 테이블/계약 |
|---|---|---|
| 인증·사용자·보안 | Yuki | `users`, `social_identities`, 동의 및 향후 세션/보안 정책 |
| 학사·강의·리뷰·졸업요건 | 도훈 | 현재/과거 강의, 교육과정, 졸업요건, 리뷰, 이수과목 |
| 시간표·자동편성 | 준서 | 향후 시간표 및 추천 작업 테이블 |

## 경계

- 현재 identity 마이그레이션은 학사 FK를 안정적으로 연결하기 위한 **공통 최소 DB 계약
  초안**입니다. 서비스 사용자 UUID와 소셜 공급자의 `(provider, provider_subject)` 식별자를
  연결하지만 실제 로그인을 구현하지는 않습니다.
- 소셜 공급자 목록, 계정 연결·해제, 토큰 저장 여부, 로그인 세션, 탈퇴·보안 정책은
  Yuki 리뷰와 구현 후 확정합니다.
- 이 브랜치는 준서 영역인 시간표·즐겨찾기·공유·추천 작업 테이블을 만들지 않습니다.
- DB 구조 변경은 세 명이 함께 합의하고, 작성자 외 최소 한 명이 PR을 리뷰합니다.
- 인증·인가 또는 개인정보 열이 바뀌면 Yuki가 최종 확인합니다.

## 병합 전 확인 책임

- 도훈: 학사 테이블, 데이터 범위, 적재·검증 재현성 확인
- Yuki: `users`, `social_identities`, `privacy_consents`와 개인정보 경계 확인
- 준서: 향후 시간표 FK가 현재 강의·분반 키를 참조할 수 있는지 확인

DB 브랜치가 기술 검증을 통과했더라도 위 공통 계약은 담당자 리뷰 없이 확정된 것으로
간주하지 않습니다.
