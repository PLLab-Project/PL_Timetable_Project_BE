# Database ownership

| 영역 | 주 담당 | 테이블/계약 |
|---|---|---|
| 인증·사용자·보안 | Yuki | `users`, `social_identities`, OTP, 동의, 세션·보안 정책 |
| 학사·강의·리뷰·졸업요건 | 도훈 | 단과대·학과/전공, 현재/과거 강의, 교육과정, 졸업요건, 리뷰, 이수과목 |
| 시간표·자동편성 | 준서 | 시간표 및 자동 편성 작업·결과 테이블 |

## 경계

- identity 마이그레이션은 서비스 사용자 UUID, 정규 학과 FK, 소셜 공급자의
  `(provider, provider_subject)` 식별자 계약을 제공합니다. 현재 로그인 구현은 학교
  이메일 OTP이며 소셜 로그인은 아직 연결되지 않았습니다.
- 소셜 공급자 목록, 계정 연결·해제, 토큰 저장 여부, 로그인 세션, 탈퇴·보안 정책은
  Yuki 리뷰와 구현 후 확정합니다.
- 시간표·자동 편성 테이블은 구현되어 있으며 즐겨찾기·공유는 아직 구현되지 않았습니다.
- DB 구조 변경은 세 명이 함께 합의하고, 작성자 외 최소 한 명이 PR을 리뷰합니다.
- 인증·인가 또는 개인정보 열이 바뀌면 Yuki가 최종 확인합니다.

## 병합 전 확인 책임

- 도훈: 학사 테이블, 데이터 범위, 적재·검증 재현성 확인
- Yuki: `users`, `social_identities`, `privacy_consents`와 개인정보 경계 확인
- 준서: 시간표·자동 편성 FK와 현재 강의·분반 키의 일관성 확인

DB 브랜치가 기술 검증을 통과했더라도 위 공통 계약은 담당자 리뷰 없이 확정된 것으로
간주하지 않습니다.
