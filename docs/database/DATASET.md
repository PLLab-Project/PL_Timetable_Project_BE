# Academic dataset

## 포함 범위

| 영역 | 확인된 범위 | 적재 행 수 |
|---|---|---:|
| 현재 강의 카탈로그 | 2026-1 | 강의 1,047 / 분반 1,576 / 수업시간 1,947 / 강의실 325 |
| 학과·전공 기준정보 | 2016~2026 | 단과대 9 / 공식 코드 171 / 요건 파생 코드 88 / 별칭 901 |
| 현재 분반 학과 연결 | 2026-1 | 명시적 학과 연결 929 |
| 과거 강의 개설 이력 | 2020~2026 | 분반 20,031 |
| 학번별 교육과정 필수과목 | 2016~2026 | 학과·전공 규칙 712 / 필수과목 2,944 |
| 학과·학번별 졸업 학점 | 2020~2026 | 학점 프로필 789 |
| 학과별 졸업 인증 | 2026 자료 | 프로필 66 / 범주 264 / 자격 세부 42 |
| 구 졸업시험·대체요건 | 보존 자료 | 규칙 35 |

테이블별 검증 기준은 `data/database/expected-row-counts.tsv`에 있습니다. 이 파일은
사용자 계정·리뷰·이수과목처럼 실행 중 증가하는 테이블을 제외하고, 39개 기준 데이터 및
적재관리 테이블만 검증합니다.

현재 분반 1,576개 중 929개에는 원천 데이터가 명시한 학과 코드가 연결됩니다. 나머지
647개는 교양·공통 강의 또는 원천에 학과 문맥이 없는 분반을 포함하므로 추측으로 학과를
채우지 않습니다.

과거 교육과정·졸업요건에만 존재하고 2020~2026 공식 학과 코드 목록과 이름으로 일치하지
않는 88개 키는 `REQ-` 파생 코드로 보존합니다. 이를 현재 공식 학과에 임의 통합하지 않고,
교육과정 필수과목·졸업 학점·졸업인증·구 졸업요건의 모든 학과 키를 FK로 연결합니다.

## 출처와 추적성

기준 데이터에는 가능한 범위에서 다음 정보를 보존했습니다.

- 데이터셋 ID와 종류
- 원본 상대 경로
- 원본 또는 정규화 결과의 SHA-256
- 문서 페이지·표·행 등 source locator
- 정규화 전후 판단에 필요한 원천 payload

원본 PDF/HWP 바이너리는 애플리케이션 실행과 DB 재현에 필요하지 않아 이 저장소에
중복 포함하지 않았습니다. 원문을 다시 파싱하거나 정규화 규칙을 재검증하려면 별도로
보관된 원본 자료가 필요할 수 있습니다.

## 자동 판정이 가능한 범위

현재 데이터는 강의 이력, 학점, 필수과목, 교양요건, 학과별 졸업인증 규칙을 조회하거나
자동 판정하는 입력으로 사용할 수 있습니다. 그러나 다음 항목은 DB 데이터만으로 최종
판정할 수 없습니다.

- 상담 횟수와 상담 완료 여부
- 편입·교환·대체과목의 개별 학점 인정
- 학과장·위원회 예외 승인
- 공학인증 참여·포기 상태
- 장애학생 면제 및 기타 개인별 예외
- 자격증·취업·수상 증빙의 유효성
- 학교 포털의 최종 졸업사정 결과

일부 규칙의 조건에 2012 같은 연도가 등장할 수 있지만, 이는 수집된 규칙의 적용 하한을
표현한 값입니다. 모든 2012학번의 학과별 교육과정 자료가 있다는 의미는 아닙니다.

## 명시적으로 제외한 데이터

- OTP challenge, 인증 rate event, 로그인 session
- 운영 중 생성되는 사용자 계정과 개인정보
- 최적화 작업 이력
- 저장 시간표와 공유 링크
- 준서 담당 시간표·추천 도메인 데이터

## 파일 구성

- `academic-data-bundle.tar.gz`: 담당자가 한 파일로 전달하는 학사 데이터 번들(Git 제외)
- `reference-data.sql.gz.part-*`: 번들에서 자동으로 풀리는 과거 강의·교육과정·졸업요건 기준 데이터
- `current-catalog.sql.gz`: 번들에서 자동으로 풀리는 2026-1 현재 학기 카탈로그
- `SHA256SUMS`: 위 SQL 적재 페이로드 파일의 무결성 값
- `manifest.json`: 패키지 ID, 생성 시각, 범위, 데이터 버전과 전체 gzip 체크섬
- `expected-row-counts.tsv`: 적재 후 기대하는 기준 테이블 행 수

분할은 내부 적재 구현에만 남아 있으며 설치자는 직접 다루지 않습니다.
`bootstrap-school.sh`가 단일 번들의 파일 목록을 제한하고 자동으로 푼 뒤, 각 조각과
결합된 전체 gzip의 SHA-256을 확인해 PostgreSQL로 스트리밍합니다. 교수명과 원천
강의계획서 payload를 포함할 수 있는 번들은 공개 저장소에서 재배포하지 않습니다.
